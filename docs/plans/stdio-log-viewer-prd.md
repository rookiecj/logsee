# stdio 로그 뷰어 PRD (Go)

## 요구사항 관리 (필수)

**모든 구현 요구사항·동작 변경은 본 PRD에 먼저 반영한 뒤** 코드·테스트를 맞춘다. 채팅·이슈만으로 확정하지 않으며, 본 문서의 표·절·검수 목록을 구현의 단일 기준으로 삼는다.

**요구사항 요약(본 문서와 불일치하는 구현·도움말·CLI 설명은 잘못된 것으로 본다)**:

- **로그 목록 화면**: **`Enter`** 또는 **`:`** → **필터 입력 모드** 진입(§6.1). **`/`** → **검색(highlight) 입력** compose 진입(§6.1·§6.5). **검색 compose 중** **`Enter`** → 확정, **`/`** 또는 **`:`** → compose 종료 후 **필터 입력**으로 전환(§6.5 표).
- **북마크(목록 탐색 전용)**: **`검색(highlight) 입력이 아닐 때`** **`m`** 과 **`1`–`9`** 로 슬롯 **9개**까지 지정·해제·점프(§6.7). **9칸 가득 후** 새 북마크는 **슬롯 1→…→9** 순으로 **순환 덮어쓰기**. **레코드 `Seq`** 기준·**`state.json` 비영속**·점프 대상이 현재 목록에 없으면 **무동작**.
- **하이라이트 매칭 이동(`n`/`p`·`Ctrl+n`/`Ctrl+p`)**: 줄·토큰 매칭은 **대소문자를 구분**한다(§8.2). **`--ignore-case`** 는 **필터·태그**에만 적용한다(§3 CLI 표).

## 확정 사항 (구현 기준)

| 태그 매칭 | **§7**: 예약 태그 **`level`** 은 **§7.1** 심각도 추출·정규화 value 비교. 그 외 태그는 **§7.4** **KV 동치**(줄 안 `tag=value` 또는 `tag: value` / `tag:value`; **로그**에서는 키·`=`/`:`·값 사이 **공백·탭 허용**). **필터** 태그 토큰에서는 **`tag` 직후 첫 `:`** 만 구분자로 인정하고 **`tag` 와 `:` 사이 공백은 지원하지 않음**(§7.3). **동일 `tag` 키**의 여러 `+` 값은 **OR**, **서로 다른 `tag` 키**·**비태그 토큰**은 **AND**. 동일 `(tag, value)` 충돌은 **§7.2** 나열 순서·선등장. **최상위 가지 OR**(`|` 단독 토큰)·`tag:` 값 병합과 `|`의 예외는 **§7.2 (6)(7)**. |
| 필터 vs 검색 하이라이트 | **권장안 A**: **로그 목록 화면**에서 **`Enter`** 또는 **`:`**는 **필터 입력 포커스**로 전환(§6.0·§6.1). **`/`**는 **검색(highlight) 입력**을 **push**(스택); compose 중에만 **검색 초안**이 편집되고, **compose 중 `Enter`**로 **`TrimSpace` 후 적용 검색어**에 반영·**compose pop**(§6.5·§8.1). **확정 전** 화면 하이라이트는 **적용 검색어만** 따른다. 확정된 적용 검색어가 비어 있지 않으면 **하이라이트 모드**이며 **`n`/`p`(로그 목록) 또는 `Ctrl+n`/`Ctrl+p`가 다음/이전 매칭 줄 이동**(§8.4). compose 중 **`Esc`**는 **목록 선택**(연속 구간 + Space pick)이 있으면 **그것만 먼저 해제**하고, 없을 때만 초안을 **적용 검색어로 되돌리며** compose만 pop한다(§6.5). **필터와의 결합**은 **스택**. |
| 필터·하이라이트 스택 | **하이라이트 모드**는 **기본 탐색 모드**(적용 필터 없음)와 **필터 모드**(적용 필터 있음) **모두**에서 동작한다. **적용 순서(레이어 스택)**: **(1) 필터**로 목록에 남길 줄 집합을 정한 뒤, **(2) 그 줄들에만** **적용 검색어** 기준 하이라이트·**`n`/`p`·`Ctrl+n`/`Ctrl+p` 매칭 이동**(목록 끝/처음에서 **순환하지 않음**)을 적용한다. 필터가 없으면 (1)은 **전체 줄**이다. 두 축은 **서로 덮어쓰지 않고 위에서 아래로 쌓인다**(필터가 검색어를 지우지 않고, 검색이 필터 식을 바꾸지 않음). |
| 검색 이동 | **하이라이트 모드**(적용 검색어가 비어 있지 않음)일 때 **`n`/`p`**(로그 목록) 또는 **`Ctrl+n`/`Ctrl+p`**는 **현재 목록**(스택의 레이어 1 결과, §8.0) 안에서 **하이라이트 기준 다음/이전 매칭 줄**로 커서만 이동한다(**비순환**, §8.4). **매칭 줄**은 **검색 토큰 중 하나라도** 부분 일치하면 된다(**OR**, §8.2; **대소문자 구분**). 적용 검색어가 비어 있으면 해당 키는 **키만 소비**. **검색 초안**은 **검색(highlight) 입력 중**에만 편집된다(§6.5·§8.1). **필터 입력 포커스**에서는 **필터 초안**이 §6.4를 따르며, 매칭 이동은 **`Ctrl+n`/`Ctrl+p`**만(초안에 `n` 입력 가능). |
| 검색 하이라이트 식 | 확정 **적용 검색어**는 **다중 토큰**으로 해석한다(§8.2): 공백·탭 구분, **`"…"`** 로 공백 포함 한 토큰, 토큰화 실패 시 **문자열 전체**를 한 바늘로 사용. **화면 강조**는 토큰별 모든 일치 구간(겹침 병합), **`n`/`p`(목록)·`Ctrl+n`/`Ctrl+p`** 줄 이동은 토큰 **OR** 매칭(줄·토큰 비교는 **대소문자 구분**, §8.2). **검색(highlight) 입력 중**에는 **`←`/`→`가 검색 초안 캐럿**에만 쓰이고(§6.5), 그때는 목록 **가로 스크롤**에 쓰이지 않는다(§6.3과 구분). **필터 입력 포커스**에서는 **`←`/`→`가 필터 초안 캐럿**에만 쓰이고(§6.4), 역시 목록 **가로 스크롤**에 쓰이지 않는다. compose·필터 입력 중에도 **`↑`/`↓` 등 §6.2·§6.3 이동키**는 **목록**을 조작한다. |
| **`--ignore-case` 범위** | 켜면 **필터·태그** 매칭(§7)에만 대소문자 무시를 적용한다. **검색 하이라이트·하이라이트 모드 `n`/`p`·`Ctrl+n`/`Ctrl+p`**에는 적용하지 않으며, 이 경로는 **항상 대소문자 구분**(§3·§8.2). |
| 커서·뷰 | **커서 줄**은 항상 뷰포트 안에 보이게 한다. **세로 스크롤은 최소 변경**: 커서가 이미 가시 영역 안에 있으면 `scrollTop`/`scrollSegTop`을 바꾸지 않는다. 커서가 **화면 첫 행**에서 **`↑`**(또는 그에 준하는 한 줄 위 이동)을 하면 **한 줄 위로** 스크롤하고, **화면 마지막 행**에서 **`↓`**를 하면 **한 줄 아래로** 스크롤한다(§8.5). 필터 줄 수 ≤ 뷰 높이일 때는 **위쪽 패딩**으로 로그를 아래에 붙여 **화면 마지막 행 = 마지막 로그 줄**. **꼬리 자동 스크롤(`follow`)**: **커서 이동키**를 쓰면 우선 **`follow`는 끊긴다**(§6.2·§8.5). 비확장 **`↑`/`↓`** 로 커서가 **실제로 이동**한 뒤 **마지막 논리 줄**이고 **꼬리 정렬**이면 **`follow`를 다시 켠다**. **예외**: 이미 **마지막 논리 줄**에서 **`↓`** 가 **커서를 움직이지 못할 때**, **`follow`가 이미 on**이고 **꼬리 정렬**이면 **`follow`를 끄지 않는다**. `PageDown`/`Ctrl+F`는 처리 후 **꼬리 정렬**이면 `follow`를 켠다(커서가 이미 마지막 줄이라 **무동작**이어도 동일). **wrap off**에서 **목록 가로 스크롤**(§6.3의 **`←`/`→`·`Shift+←`/`Shift+→`**, **필터 입력 포커스의 `←`/`→` 제외**, §6.4) 후에도 **꼬리 정렬**이면 **`follow`를 다시 켠다**(꼬리가 아니면 끈다). `End`/`Shift+End`는 맨 아래로 가며 **항상 `follow`를 켠다**. |
| 줄 줄바꿈(wrap) | **`Ctrl+W`**로 **줄 줄바꿈(line wrap)** on/off(목록·필터 입력 포커스 공통, §6.1). **기본(off)**: 긴 논리 줄은 **가로 스크롤**만(§5). **on**: 본문을 터미널 너비(시퀀스 컬럼 제외)에 맞춰 **여러 시각 행**으로 나누고, 이어지는 행의 시퀀스 자리는 **공백 패딩**으로 맞춘다. **wrap on으로 전환할 때** 가로 오프셋은 **0**으로 리셋한다. **세로 스크롤·꼬리 정렬·짧은 목록 패딩**은 **시각 행** 수를 기준으로 한다(구현상 `scrollSegTop`). 커서·선택·follow·`PageUp`/`PageDown`의 “줄” 단위는 **논리 줄(필터 목록 인덱스)** 을 유지한다(§5·§8.5). **`follow`는 wrap on/off와 무관**하게 동작한다(꼬리 정렬·신규 줄 자동 스크롤 포함; wrap on일 때 시각 꼬리 맞춤은 §8.5). **wrap on**일 때 **로그 목록 화면**에서 §6.3의 `←`/`→`·`Shift+←`/`Shift+→`는 **가로 오프셋을 바꾸지 않는다**. **필터 입력 포커스**의 `←`/`→`는 §6.4대로 **필터 초안 캐럿**만 바뀐다. |
| 줄 선택 | **연속 구간(선택영역 설정)**: **Shift+↑↓**(주) / **Shift+Home/End**(보조)로 앵커~커서 구간. **Space pick**: **로그 목록 화면**이고 **검색(highlight) 입력이 아닐 때** **`Space`**로 현재 커서 줄을 pick 집합에 **토글**; **연속 구간이 활성일 때 `Space`**는 그 구간의 **모든 줄**을 pick에 **넣은 뒤** 연속 구간(앵커)만 **해제**한다. **선택 모드**: 연속 구간이 있거나 pick이 **1줄 이상**이면 진입한 것으로 본다. **`c`**: **연속 구간**에 속한 줄과 **Space pick**된 줄의 **합집합**(같은 필터 목록 인덱스는 **한 번만**)을 필터 목록 인덱스 **오름차순**으로 개행 연결해 클립보드에 복사한다. **구간도 없고 pick도 없으면** **커서 한 줄**만 복사한다(로그 목록·compose 아님·목록 비어 있지 않음); 성공 시 **`N lines copied`** / **`1 line copied`**. **`Esc`** 목록 첫 단계: 연속 구간과 pick을 **한 번에** 해제(§6.5·§6.6). **비확장 세로·페이지·Home/End·가로 스크롤·`n`/`p`·`Ctrl+n`/`Ctrl+p`** 등은 **연속 구간만** 해제하고 **pick은 유지**; **로그 목록에서 `Enter` 또는 `:`(필터 입력 진입)**·필터 적용 성공 등은 **전부** 해제(§6.3·§8.6). |
| 파일 출력 | **입력이 stdin**(인자 없음 또는 `-`)일 때만: `--out` 생략 시 **현재 작업 디렉터리**에 `logsee-YYYYMMDD-HHMMSS.log`(초 단위 시각)로 자동 생성·append. 같은 초에 여러 번 실행하면 **동일 파일**에 이어 붙는다. 지정 시 해당 경로에 원문 줄 append. **크기 회전**: 기본 **`--out-max-bytes`(10 MiB)** … **`--out-max-bytes 0`** 이면 회전 없음. 수신 **한 줄마다** `--out`에 **쓰기 버퍼 없이** 직접 기록하고 **`Sync()`까지 수행한 뒤** 링 버퍼·UI에 반영한다. **입력이 `input-file`**(경로 인자)일 때는 **원본이 이미 파일**이므로 **`--out`을 열지 않고** 줄을 **디스크에 다시 쓰지 않는다**(상태바 `out:-`). 링·UI 반영만 한다. **`--out` 플래그는 이 모드에서 무시**된다. |
| TUI 스택 | **bubbletea** + **lipgloss** (리사이즈·오버레이·메시지 루프에 적합). **`F1`** 인앱 도움말(버전·키맵·**필터 입력 중 필터 문법 요약**, §5·§6.4). |
| 키보드 입력 | 파이프 사용 시 `stdin`은 로그 전용이므로 **`/dev/tty`**를 bubbletea 입력으로 사용. |
| 이동키 | **로그 목록 화면**(`FocusLogList`)에서 **`h`/`j`/`k`/`l`** 은 **`←`/`↓`/`↑`/`→`** 와 동일하게 **목록**만 이동하고, **`G`** 는 **`End`** 와 동일(마지막 줄·`follow` 켬, §6.2·§6.3). **필터 입력·검색(highlight) compose** 중에는 **해당 초안에 인쇄 문자**로만 입력된다(§6.4·§6.5). |
| 필터·하이라이트 히스토리·영속화 | **마지막 적용 필터 문자열**·**마지막 적용 검색어**와 각각의 **MRU 목록**(최대 **50**개, 중복 시 **맨 앞으로** 이동)을 **JSON**으로 저장한다. 기본 저장 디렉터리는 **`$HOME/.local/logsee/`**이며 그 아래 고정 파일명 **`state.json`** 을 쓴다. **`--state-dir`** 로 디렉터리를 바꿀 수 있다(비어 있으면 기본). **앱 시작 시** 파일이 있으면 **마지막 적용값**을 복원해 필터·하이라이트에 반영한다; **저장된 필터 문자열이 현재 파서에서 오류**이면 **한 줄 `stderr`** 후 **필터 없음**으로 시작한다. **필터 입력 모드**에서 **`↓`**(히스토리 오버레이가 **닫혀 있을 때**)는 **로그 목록을 움직이지 않고** **필터 히스토리 오버레이**를 연다(§6.4.1). **검색(highlight) 입력 중**에도 **`↓`**(오버레이 **닫힘**)는 **하이라이트 히스토리 오버레이**를 연다(§6.4.1). 오버레이 **열림** 중 **`↑`/`↓`**는 **목록 내 선택만** 이동한다. **`Esc`**: 오버레이만 닫고 **현재 초안 문자열은 유지**한다. **`Enter`**: 선택 항목을 **해당 초안**에 반영하고 오버레이를 닫는다(빈 히스토리면 변화 없이 닫을 수 있음). 오버레이가 열린 동안 **그 외 키는 소비만** 한다. **히스토리 목록에 넣는 시점**: 필터는 **`Enter`로 파싱 성공·적용**된 문자열이 **공백만이 아닐 때** MRU에 넣는다(빈 식으로 필터를 끈 경우 MRU에는 넣지 않음). 하이라이트는 **compose 중 `Enter`로 확정**된 적용 검색어가 **비어 있지 않을 때만** MRU에 넣는다. **파일 쓰기**는 **적용 직후**(필터 `Enter` 성공·검색 compose `Enter` 확정)에 수행한다. |
| 북마크 | **§6.7**: 고정 슬롯 **1–9**, **`m`**(할당·해제)·**`1`–`9`**(해당 슬롯으로 커서 이동). **9칸 가득**이면 **`m`** 추가 시 **1→…→9→1** **순환 덮어쓰기**. **저장 값은 레코드 `Seq`**(필터 인덱스가 아님). **`state.json`에 저장하지 않음**. **로그 목록**이고 **검색(highlight) 입력이 아닐 때**에만 북마크 규칙이 적용된다; **검색 compose 중**에는 **`m`·`1`–`9`는 §6.5대로 검색 초안에 삽입**된다. |

## 1. 제품 개요

**목적**: 파이프/리다이렉션으로 들어오는 텍스트 로그를 실시간으로 목록에 표시하고, **stdin 입력일 때만**(기본 경로 또는 `--out`) 파일에 원문을 누적 저장하며, **파일을 직접 열어 읽을 때는 저장하지 않는다**, 키보드로 탐색하고 필터·검색 하이라이트로 원하는 줄을 찾는 **터미널(TUI) 도구** (`logsee`).

**예시**: `some-cmd 2>&1 | logsee` · `some-cmd 2>&1 | logsee --out session.log`

**플랫폼**: macOS/Linux, UTF-8. Windows는 2차.

## 2. 용어

| 용어 | 정의 |
|------|------|
| **레코드** | 개행으로 구분된 한 줄 + 단조 증가 **시퀀스 번호**. |
| **뷰포트** | 필터 적용 후 목록에서 보이는 연속 구간. |
| **필터** | 표시 대상 레코드를 줄이는 토큰 집합. **비태그 토큰**과 **서로 다른 태그 키**는 **AND**; **동일 태그 키**의 여러 `+` **value**는 **OR**(§7). |
| **필터 입력 모드** | **로그 목록 화면**에서 **`Enter`** 또는 **`:`**로 진입한 뒤 **적용 전** 필터 식을 편집하는 상태. **첫 줄 필터 크롬**에 **`FILTER  │  > …`** 형태로 **필터 초안**을 표시하고(캐럿 위치는 §5), 키보드 **필터 입력 포커스**가 이 줄에 있다(§6.0). |
| **로그 목록 화면** | **필터 입력 모드가 아닐 때**. 키 입력이 **로그 목록**(커서·스크롤·**검색(highlight) 입력**·`/`·`Enter`·`:` 등)에 우선 적용된다. |
| **필터 입력 포커스** | **필터 입력 모드**일 때. **`←`/`→`는 필터 초안 캐럿**만 좌·우로 이동(§6.4)하며, **목록 가로 스크롤**(§6.3)에는 쓰이지 않는다. 인쇄 문자·`Space`·`Backspace`는 **필터 초안**(캐럿 위치)에 적용되며, **§6.2 공통 이동키**와 **§6.3** 표의 **그 밖의** 키는 초안에 넣지 않고 목록만 조작한다. |
| **필터 모드** | `Enter`로 파싱에 **성공**한 뒤 **적용 필터 문자열이 비어 있지 않은** 상태. 목록은 필터를 통과한 줄만 보이며, **필터 입력 모드가 아니다**. 적용 식을 공백만으로 두면 파싱 결과가 “필터 없음”과 동일(전체 줄 표시). |
| **기본 탐색 모드** | 적용 필터가 **없고** **필터 입력 모드**도 아닌 상태. 목록 후보는 **링에 있는 전체 줄**이다. 이 상태에서도 **하이라이트 모드**(적용 검색어가 비어 있지 않음)는 그대로 동작한다(§8.0). |
| **검색(highlight) 입력** | **로그 목록 화면**에서 **첫 `/`**로 진입하는 한 겹 스택. 이 동안만 **검색 초안**을 편집한다. **compose 중 `Enter`**로 확정·pop, **`Esc`**는 **목록 선택**(구간·pick)이 **없을 때만** 초안을 **적용 검색어로 되돌리고** pop(§6.5). |
| **검색 초안** | **검색(highlight) 입력 중**에만 편집되는 문자열(둘째 크롬 줄 `HIGHLIGHT  │  > …`; 캐럿이 끝이면 `…_`, 중간이면 `…|…` 형태로 표시, §5). **캐럿 위치**에 `Backspace`·`Space`·인쇄 문자가 반영된다(**매칭 이동은 `Ctrl+n`/`Ctrl+p`**, §6.3). **`←`/`→`는 캐럿만 이동**(§6.5). **compose 중**에는 **`Space`**가 검색 초안에만 들어가고 **목록 pick에는 영향 없음**(§6.3). **확정 시** `TrimSpace`한 값이 **적용 검색어**가 되고 초안은 그에 동기화된다(§8.1). |
| **적용 검색어** | **검색(highlight) 입력 중 `Enter`로 확정**된 문자열(바깥 `TrimSpace` 후)만 **하이라이트**·**`n`/`p`(목록)·`Ctrl+n`/`Ctrl+p` 매칭**에 사용한다(구현 필드명 `searchBuf`). **토큰 분리·따옴표·OR 매칭·대소문자 구분 부분 일치**는 §8.2. **기본 탐색 모드**와 **필터 모드** **모두**에서 유지되며, 하이라이트·**매칭 이동**은 **항상 현재 목록(필터 통과 후)** 위에만 적용된다(**스택**, §8.0). |
| **키보드 Focus** | 한 번의 키 입력에서 **일반 키**를 소비하는 **비모달** 주체. 구현은 `FocusLogList` \| `FocusFilterEditor` \| `FocusHighlightEditor`(§6.0.1). **`F1` 도움말**·**§6.4.1 히스토리 오버레이**는 열린 동안 **모달 최상위**로, 본 Focus보다 **먼저** 라우팅된다. |
| **커서** | 필터된 목록에서의 포커스 줄 인덱스. **세로 스크롤**은 커서가 뷰포트 밖으로 나가지 않게 **최소한만** 조정한다(§8.5). |
| **연속 구간(선택영역)** | `Shift+↑↓`·`Shift+Home/End`로 잡는 앵커~커서 **한 덩어리** 구간. **비확장 `↑`/`↓`** 등으로 커서만 움직이면 **이 구간(앵커)만 해제**되고 **Space pick 집합은 유지**된다(§6.2·§6.3). |
| **Space pick** | 필터 목록 인덱스 기준, **`Space`**로 토글한 **줄 집합**(비연속 가능). **로그 목록 화면**이고 **검색(highlight) 입력이 아닐 때만** 변한다(§6.3). |
| **북마크 슬롯** | **1–9**번의 **고정 슬롯** 각각이 **비어 있거나** 하나의 **레코드 `Seq`** 를 가리킨다(§6.7). **슬롯 번호는** 다른 슬롯에서 북마크를 **해제해도 재번호하지 않는다**. |
| **선택 모드** | **연속 구간이 활성**이거나 **Space pick이 1줄 이상**인 상태. **`c`**·**`Esc`** 첫 단계 등은 §6.5·§6.6·§8.6을 따른다. |
| **선택**(레거시 총칭) | UI에서 **연속 구간**과 **Space pick**을 아울러 이르는 맥락이 있으나, 동작 차이는 §6.3·§8.6 표가 우선한다. |
| **하이라이트 모드** | **적용 검색어**가 **비어 있지 않은** 상태. **기본 탐색 모드**·**필터 모드** **어느 쪽에서도** 켤 수 있으며, **필터가 있으면 필터를 통과한 줄에만**, 없으면 **전체 표시 줄**에 하이라이트가 그려진다(**레이어 스택**, §8.0). **필터 입력 모드** 중에도 이전에 확정된 적용 검색어가 있으면 동일하게 표시·**매칭 이동**이 적용된다. **`n`/`p`(로그 목록)·`Ctrl+n`/`Ctrl+p`**는 하이라이트와 동일한 매칭 기준으로 다음/이전 줄 이동에 쓴다(**필터 입력**·**검색 compose** 중에는 **`n`/`p`는 각 초안 문자**, 매칭 이동은 **`Ctrl+n`/`Ctrl+p`**만). **검색 초안**·**필터 초안** 편집은 각각 §6.5·§6.4·§8.1을 따른다. **목록**에서 `Esc` 최종 단계로 적용 검색어·검색 초안을 모두 비우면 하이라이트 모드가 해제된다(§6.5·§6.6). |

## 3. CLI

```
logsee [flags] [input-file]
```

- **input-file** 생략: **stdin**에서 로그를 읽는다(파이프·리다이렉션).
- **input-file** 지정: 해당 **파일**을 처음부터 읽어 EOF까지 스트리밍한다. 성장하는 로그는 `tail -f file | logsee`처럼 stdin으로 넘기면 된다.
- **`"-"`**: 명시적으로 stdin을 쓴다.

| 플래그 | 기본 | 설명 |
|--------|------|------|
| `--out` | 자동(stdin만) | **stdin 입력**일 때만 적용: 비어 있으면 `logsee-YYYYMMDD-HHMMSS.log`(현재 디렉터리, 초 단위). 지정 시 그 경로에 수신 **원문** 줄 append. **`input-file`로 읽을 때는 무시**(출력 파일 없음). |
| `--out-max-bytes` | `0` (회전 비활성, 단일 파일) | 플러시된 `--out` 크기에 **다음 한 줄**을 더했을 때 이 값을 **초과**하면 **크기 회전**(확정 표·요약 표). 기본 `0` 은 회전 비활성(단일 파일) — stdin scrollback 이 링 용량을 넘어 전체 파일을 커버한다. `>0` 설정 시 회전 발생하며, 회전 이후 seq 범위까지만 scrollback 가능. |
| `--max-lines` | `100000` | 링 버퍼 최대 줄 수. |
| `--ignore-case` | off | **필터·태그** 매칭 시 대소문자 무시. **검색 하이라이트·하이라이트 모드 `n`/`p`·`Ctrl+n`/`Ctrl+p`**는 **항상 대소문자를 구분**하며(§8.2), 이 플래그의 영향을 받지 않는다. |
| `--no-line-numbers` | off | 시퀀스 컬럼 숨김. |
| `--sync-interval` | `0` | **`--out`이 열릴 때(stdin 입력)** 만: `>0`이면 주기적 `fsync` (예: `1s`). `0`은 flush 정책만. **stdin**에서는 줄마다 `Sync()`이므로 이 플래그는 사실상 **`--out` 회전·flush 정책**에만 관여한다. **`input-file` 입력**(출력 없음)일 때는 **무의미**한다. |
| `--stdin-batch-ms` | `40` | 로그 입력 줄을 **UI 갱신**만 묶는다(stdin·파일 읽기 공통). **`--out` 기록은 배치하지 않으며** stdin 경로에서는 줄마다 즉시 디스크 반영(확정 표·§4). `0`이면 UI도 줄마다 즉시 갱신. |
| `--state-dir` | `$HOME/.local/logsee` | **필터·하이라이트 히스토리** JSON(`state.json`)을 둘 **부모 디렉터리**. 디렉터리는 없으면 **저장 시** 생성한다. |
| `--config` | `$HOME/.local/logsee/config.json` | 로그 타입/패턴 설정 JSON 경로. 파일이 없으면 기본값으로 동작. |
| `--log-type` | `auto` | 예약 태그 **`level`** 추출에 쓰는 줄 형태: **`plain`**(구조적 패턴 없음), **`adb`**(Android logcat `-v time` 형), **`auto`**(아래 probe로 adb/bracket 점수 판별·android 우세일 때만 adb·그 외는 **plain**). 별칭: `android` = `adb`. |
| `--log-type-probe-lines` | `32` | **`--log-type=auto`** 일 때만: 링의 **가장 오래된** 줄부터 **비어 있지 않은** 줄을 최대 이 개수까지 샘플해 포맷을 한 번 확정한다(§7.1). `≥1`. 샘플이 부족하면 입력이 끝날 때(EOF)까지 기다렸다가 그 시점에 확정한다. |

종료: `q` 또는 `Ctrl+C` (버퍼 flush 후 종료).

## 4. 입력·줄 처리

- **입력**: 인자 **없음** 또는 **`"-"`**이면 `os.Stdin`에서 읽음. **positional `input-file`**이면 해당 경로의 파일을 읽음(일반 파일만; 디렉터리 등은 오류). 키보드는 여전히 **`/dev/tty`**.
- **`--out`과 입력 소스**: **stdin**이면 확정 표대로 **줄마다 무버퍼 쓰기 + `Sync()` 후** 링·UI. **파일 입력**(`input-file`)이면 **`--out` 없이** 링·UI만 갱신(원본 파일에 다시 쓰지 않음); **`--out` 플래그는 무시**된다.
- **줄 분리**: **LF**(`\n`)으로 논리 줄을 끝낸다. **CRLF**(`\r\n`)는 한 줄 끝으로 처리한다. **단독 CR**(`\r` 뒤에 LF가 오지 않음)은 터미널과 같이 **현재 줄을 버퍼 맨 앞으로 되돌려** 이후 문자가 앞부분을 덮어쓴다(진행률 한 줄 갱신 등). **파일이 `\r`로만 끝남**(Mac 고전): 그 앞까지를 한 줄로보낸다. 구현은 [internal/loginput/scan.go](internal/loginput/scan.go)의 `ScanLines`.
- **인코딩**: UTF-8. 잘못된 바이트 시퀀스는 수신 직후 `strings.ToValidUTF8(..., "\uFFFD")`로 **U+FFFD 치환** 후 버퍼·(**`--out` 사용 시** 파일)·UI에 반영한다.

### 4.1 입력 파일 부분 로딩(대용량)

- **적용**: **positional `input-file`**(stdin 아님)일 때만. **stdin**은 기존과 같이 스트림으로 받아 링에 누적한다.
- **초기 로드**: 파일의 **앞쪽부터** 논리 줄을 읽어, **약 2×뷰포트 높이(`viewportH`)** 만큼(구현: 최대 42줄)만 메모리에 올린다. 디스크 읽기는 **4 KiB 청크**로 하며, 줄 경계는 [internal/loginput/scan.go](internal/loginput/scan.go)와 동일한 규칙으로 복원한다.
- **인덱스**: UI 표시 직후, 백그라운드에서 **전체 파일의 각 논리 줄 시작 바이트 오프셋** 테이블을 한 번 스캔해 구축한다([internal/loginput/scan.go] `LineStartOffsets`). 완료 후 총 줄 수가 확정된다.
- **시퀀스(`Seq`)**: 파일 모드에서는 `Seq`가 파일 내 **1-based 줄 번호**이며, 링 버퍼는 **현재 로드된 윈도우**만 담는다. stdin 모드의 도착 순서 `Seq`와 의미가 다르다(파일 모드는 줄 번호 고정).
- **초기 커서·follow**: 부분 로딩 시작 시 **커서는 첫 줄**, **`follow`는 off**로 둔다.
- **윈도우 이동**: 사용자가 **↑/↓**, **PageUp/PageDown**, **Home/End** 등으로 **로드된 범위 밖**으로 가려 하면, 대상 줄을 중심으로 **약 2×`viewportH` 줄**을 다시 읽어 버퍼를 교체한다(양방향).
- **하이라이트 매칭 이동과 부분 로딩**: 파일 부분 로딩 모드에서 **`n`/`p`·`Ctrl+n`/`Ctrl+p`** 탐색 시, 현재 윈도우에 매치가 없으면 **같은 방향으로** 추가 윈도우를 읽어 매치를 계속 찾는다(비순환 유지). 탐색 중 누적 디스크 읽기량이 **100 MiB**를 넘기면 **중앙 확인 모달**로 계속 진행 여부를 묻고, 취소 시 현재 커서를 유지한다. 매치가 발견되어 윈도우를 다시 잡을 때는 **`n`(다음)** 은 매치가 화면 **하단 쪽**에 오도록, **`p`(이전)** 는 매치가 화면 **상단 쪽**에 오도록 배치한다.
- **필터**: 적용 필터는 **현재 메모리에 올라온 줄**에만 적용된다(전 파일 스캔 필터는 본 절 범위 밖).
- **필터 보강 로딩**: 파일 부분 로딩 모드에서 필터 적용 직후 현재 윈도우의 매치 수가 **뷰포트 높이보다 작으면**(`len(matches) < viewportH`) **앞 방향으로** 추가 윈도우를 순차 로드하며 매치를 계속 찾는다. 한 화면을 채울 만큼 확보되면 중단하고, 끝까지 없으면(최종 0건) `no lines`로 유지한다.
- **북마크(§6.7)**: 슬롯에 저장된 `Seq`가 현재 윈도우에 없으면, 인덱스가 준비된 뒤 해당 줄이 포함되도록 **윈도우를 먼저 로드**한 다음 커서를 이동한다. 인덱스가 아직 없으면 해당 점프는 무동작과 동일하게 할 수 있다.

## 5. UI

- **상태바**: 필터 요약, 필터 오류(있을 때), **입력**(`in:stdin:read` / `in:<path>:read` · EOF 후 `:eof`), 레코드 수, **`type:`** — 로그 타입(§3 `--log-type`·§7.1): `type:plain` / `type:adb` 또는 **`auto` 확정 후** `type:adb` 등; **`auto`이고 아직 probe가 부족**하면 `type:auto~`, follow 여부, **출력**(`out:<path>` · **stdin·저장 사용 시** / `out:-` · **파일 입력·저장 없음**), 복사 결과 피드백(`N lines copied` / `1 line copied` / `no lines` 또는 오류 한 줄; **토스트성**으로 다음 키 입력 시 비움), 연속 구간이 있으면 **`sel:시작-끝`**(1-based), Space pick이 있으면 **`picked:N`**(줄 개수)을 **병기**(둘 다 없으면 생략). 클립보드 피드백 구간은 **밝은 전경·녹색 배경 등으로 하이라이트**(나머지 상태 필드와 시각적으로 구분)해 **시인성**을 높인다.
- **상태바 on/off 표시**: **`follow:`·`wrap:`** 등 **이진 상태**를 나타낼 때, 값 **`on`** 은 **색(예: 녹색 계열 전경)** 으로 강조하고, 값 **`off`** 는 **회색 톤(낮은 채도·흐림)** 으로 표시한다. 라벨(`follow:` 등)은 상태바 기본 전경색을 따른다.
- **도움말(F1)**: **`F1`** 으로 **도움말 다이얼로그**(터미널 중앙 모달)를 연다. **`Version:`** 줄(또는 동등 한 줄)에 **빌드 정보**를 표시한다(구현: `runtime/debug.ReadBuildInfo()`로 만든 한 줄 — 모듈 경로·`Main.Version`·가능하면 **`vcs.revision`** 앞부분). **필터 입력 포커스**(§6.4)에서 **`F1`** 이면 **필터 문법·결합 규칙·입력 예·필터 전용 키**를 담은 **필터 도움말**을 연다(전역 키맵 도움말과 별도 본문). **그 외 포커스**에서 **`F1`** 이면 기존과 같이 **축약 키맵**을 **§6의 포커스·모드**에 맞춰 **구역(섹션)을 나누어** 표시한다: **(1) 이 대화창** — 도움말이 열린 동안만 해당하는 **닫기·종료**(`F1`/`Esc`·`q`/`Ctrl+C`); **(2) 공통** — §6.1 수준의 **전역 기능**(`F1`·`q`/`Ctrl+C`·`Ctrl+W`·`Enter`·`:`·`/` 등); **(3) 로그 목록 화면** — §6.2·§6.3·**§6.7(북마크)** 및 **`Esc`** 요약; **(4) 필터 입력** — §6.4; **(5) 검색(highlight) 입력** — §6.5 표. 같은 키가 모드에 따라 다르면 **해당 모드 구역**에만 적고, 상세 우선순위·예외는 **§6 본문 표**가 우선한다. **`Esc`** 또는 **`F1`** 로 닫는다. 다이얼로그가 열려 있는 동안 **그 외 키는 소비만** 하며(로그·필터·검색·커서 상태는 유지), **`q`·`Ctrl+C`** 는 **여전히 종료**한다.
- **첫 줄(필터 크롬)**: **`FILTER  │  …`** 패턴. **필터 입력 모드가 아닐 때**는 적용 필터 문자열(또는 비어 있으면 **`∅`**)·**`Enter/:=filter edit`** 등 힌트를 병기한다. **필터 입력 모드**일 때는 **`FILTER  │  > `** 뒤에 **필터 초안**을 두고, **캐럿이 가리키는 한 루**(UTF-8)에 **Reverse(역전)** 스타일을 적어 가로 위치를 표시한다(**캐럿이 초안 맨 끝**이면 역전 **`_`** 한 칸). 터미널 너비를 넘으면 **왼쪽부터** 잘라 표시한다(캐럿이 잘린 접두 바깥이면 역전 없이 접두만 보일 수 있음). 파싱 실패 시 **`err:`** 등 기존과 같이 같은 줄에 오류 문구를 병기한다.
- **둘째 줄(하이라이트 크롬)**: 첫 줄 **필터 크롬**과 같은 패턴(**`HIGHLIGHT  │  …  │  힌트`**). **필터 입력 모드**일 때는 하이라이트 줄에 필터 적용 안내만 짧게 둔다(**`Enter`/`Esc`/`Backspace`/`←`/`→`(초안 캐럿)** 등, §6.4). **로그 목록 화면**에서 **적용 검색어(하이라이트)** 가 비어 있으면 **`∅`** 표시, 있으면 **문자열 그대로**(다중 토큰·따옴표 포함 원문) 표시하고 **`/=highlight edit`** 등 힌트를 병기한다. **검색(highlight) 입력** 중에는 필터 편집과 같이 **`HIGHLIGHT  │  > …`** 로 **검색 초안**을 두고, **캐럿이 초안 끝**이면 **`>` 뒤에 초안 + `_`**, **캐럿이 중간**이면 **캐럿 왼쪽 텍스트 + `|` + 오른쪽 텍스트**로 위치를 표시한다. 배경색은 **필터 입력 중**과 동일 계열로 강조한다.
- **목록**: 시퀀스 번호(옵션) + **북마크 마커(옵션, §6.7)** + 본문. **시퀀스 컬럼이 켜져 있으면**(`--no-line-numbers` 기본 off) **시퀀스 번호와 슬롯 숫자 사이에 공백 한 칸**을 두고, 해당 줄이 북마크 슬롯 `k`를 점유하면 **`1`–`9`의 한 자리 숫자(`k`)** 를 **배경·전경색이 있는 배지(chip)** 형태로 그려 **시퀀스 숫자와 시각적으로 분리**한다(구현: lipgloss; **커서 행**에서는 역전(Reverse)과 겹쳐도 식별되도록 **배지 색 구성을 달리**한다). **`--no-line-numbers`가 on**이면 **본문 시작 직전**에 동일 규칙으로 **두 칸 디스플레이 폭**(빈 슬롯은 공백)을 확보해 표시한다. **wrap on**일 때는 **논리 줄의 첫 시각 행**에만 시퀀스·북마크를 표시하고, 이어지는 시각 행은 §5·§6 기존 규칙대로 시퀀스 자리만 공백 패딩한다(북마크 마커도 첫 행에만). **기본(wrap off)**: 긴 논리 줄은 **가로 스크롤**(`←` `→`만; **로그 목록**에서 **`h`/`l`** 은 동일, §6.3). **`Ctrl+W`**로 **줄 줄바꿈(wrap)** 을 켜면(확정 표·§6.1) 같은 논리 줄이 **여러 시각 행**으로 표시되며, **현재 커서 논리 줄**의 **모든** 시각 행에 Reverse 등으로 동일 강조를 적용한다(**`follow` on/off**와 무관; 내부 `cursorIdx`·스크롤·§8.5 키 동작은 그대로). **연속 구간**과 **Space pick** 줄은 각각 배경 톤으로 구분(연속 구간과 pick만 겹칠 때는 연속 구간 스타일 우선); 단위는 **논리 줄**이다.
- **Follow(꼬리 자동 스크롤)**: **커서 이동**으로 follow가 꺼진 뒤(확정 표·§8.5), 커서가 다시 **마지막 논리 줄**에 있고 뷰가 **꼬리 정렬**이면 follow가 켜진다. **마지막 논리 줄**에서 **`↓`** 를 눌렀으나 **커서가 움직이지 않을 때**는, **이미 follow가 켜져 있고** 꼬리 정렬이면 **끄지 않고 유지**한다. **wrap off**에서 **가로 스크롤**만 한 경우에도 **여전히 꼬리 정렬**이면 동일하게 follow가 켜진다. **wrap on**에서도 동일하며, 시각 행이 뷰 높이를 넘으면 꼬리 정렬은 **마지막 시각 행이 화면 마지막 행**에 오도록 `scrollSegTop`을 맞춘다(§8.5). 켜진 상태에서 **새 로그 줄**이 들어오면 커서·스크롤을 갱신해 **자동으로 꼬리를 따라간다**(wrap off면 `scrollTop`, wrap on이면 `scrollSegTop`). **`End`/`Shift+End`**는 맨 아래로 가며 follow를 **명시적으로 켠다**.

## 6. 키맵

### 6.0 포커스와 `Esc`(모드 스택 pop)

- **로그 목록 화면**: **필터 입력 모드가 아닐 때**. 키 입력이 **로그 목록**(커서·스크롤·**검색(highlight) 입력**·`/`·`Enter`·`:` 등)에 우선한다.
- **필터 입력 포커스**: **필터 입력 모드**일 때. **필터 초안** 편집(`←`/`→` 캐럿·인쇄 문자 등, §6.4)이 우선하되, **`←`/`→`를 제외한** **§6.2 공통 이동키**와 **§6.3** 표의 키는 초안에 넣지 않고 **목록**만 조작한다.
- **도움말 다이얼로그**(§5): **`F1`** 로 연 상태가 **최상위**이다. **`Esc`·`F1`** 은 **다이얼로그만 닫고** 필터·검색·로그 목록 화면 규칙은 **변경하지 않는다**. **`q`·`Ctrl+C`** 는 **종료**가 우선이다.
- **`Enter`** 또는 **`:`**(§6.1, **로그 목록 화면**): **필터 입력 포커스**로 전환한다(필터 입력 모드 진입).
- **`/`**(§6.1, **로그 목록 화면**): **검색(highlight) 입력**을 연다(compose push).
- **`Esc`**: **도움말이 열려 있으면** §5대로 **닫기만** 한다. 그렇지 않으면 **모드 스택에서 한 단계 pop**. **필터 입력 포커스**이면 **로그 목록 화면**으로만 돌아간다(초안 편집 취소·`filterDraft`를 적용 필터 문자열로 되돌림, 적용 `prog`는 변경 없음). **로그 목록 화면**이면 **§6.5**(기본 탐색) 또는 **§6.6**(필터 모드)의 `Esc` 우선순위를 따른다.

### 6.0.1 키보드 Focus 타입과 `handleKey` 라우팅 (구현 단일 기준)

- **목적**: 키 입력을 **어느 주체가 먼저 받는지**를 코드·문서에서 동일한 이름으로 고정한다.
- **상수** (`internal/ui/focus.go`): **`FocusLogList`** · **`FocusFilterEditor`** · **`FocusHighlightEditor`**.
- **파생 규칙**: `filterEdit` 가 참이면 **`FocusFilterEditor`**; 아니고 `searchCompose` 가 참이면 **`FocusHighlightEditor`**; 그렇지 않으면 **`FocusLogList`**. **앱 시작 직후**는 **`FocusLogList`** (`filterEdit`·`searchCompose` 둘 다 거짓).
- **`handleKey` 순서** (동일 파일·`model.go`의 루프와 합치면): **`q` / `Ctrl+C` 종료** → **`helpOpen`이면 도움말 전용** → **`F1`** (도움말 열기; **`FocusFilterEditor`** 이면 필터 문법 도움말 플래그) → **`histKind` 오버레이면 §6.4.1** → **`Ctrl+W`** (wrap 토글) → **`FocusFilterEditor`가 아닐 때** **`Esc` + 목록 선택(구간·pick) 있음**이면 **선택만 해제**하고 반환 → **`keyFocus()` 분기** (`FocusFilterEditor` → 필터 편집·§6.4, `FocusHighlightEditor` → 검색 compose·§6.5, `FocusLogList` → 목록·§6.1–§6.3).
- **동작 예** (요구사항과의 대응): **`FocusLogList`** 에서 **`/`** → **`FocusHighlightEditor`** 로 전환(검색 compose). **`FocusLogList`** 에서 **`Enter`** 또는 **`:`** → **`FocusFilterEditor`**. **이동키**는 필터·검색 편집 중에도 **목록**으로 전달되는 경로가 §6.2·§6.4·§6.5와 같다. **`h`/`j`/`k`/`l`** 는 **`FocusLogList`** 에서만 **`←`/`↓`/`↑`/`→`** 와 동일(§6.3); **`G`** 는 **`FocusLogList`** 에서만 **`End`** 와 동일; **`FocusFilterEditor`**·**`FocusHighlightEditor`** 에서는 §6.4·§6.5대로 **초안 문자**로 처리된다. **`m`** / **`1`–`9`** 북마크는 **`FocusLogList`** 일 때만 §6.7; **`FocusHighlightEditor`** 에서는 §6.5대로 초안 문자로 처리된다.

### 6.1 공통 기능키

| 키 | 동작 |
|----|------|
| `q`, `Ctrl+C` | 종료(**도움말 다이얼로그가 열려 있어도** 동일). |
| `F1` | **도움말 다이얼로그**를 연다(§5). 이미 열려 있으면 **닫는다**. **`FocusFilterEditor`** 이면 **필터 문법·요약 도움말**을, **그 외**(`FocusLogList`·`FocusHighlightEditor`)이면 **전역 키맵 도움말**을 연다. |
| `Ctrl+W` | **줄 줄바꿈(line wrap)** on/off 토글. wrap on으로 바꿀 때 **가로 스크롤 오프셋은 0**으로 리셋한다. **로그 목록 화면**·**필터 입력 포커스** 모두에서 동작한다(§6.2와 같이 필터 초안에 문자로 들어가지 않음). **도움말이 열려 있으면** 적용되지 않는다(§5). |
| `Enter` | **로그 목록 화면**에서 **필터 입력 포커스**로 전환: **필터 입력 모드** 진입, **연속 구간 + Space pick 전부 해제**, 초안을 현재 적용 필터 문자열로 초기화. |
| `:` | **`Enter`와 동일**(로그 목록 화면에서만; 필터 입력 모드 진입·선택 해제·초안 초기화). |
| `/` | **로그 목록 화면**에서 **검색(highlight) 입력** 시작: **검색 초안**을 현재 **적용 검색어**로 초기화하고 compose를 켠다(§6.5). **연속 구간 + Space pick 전부 해제**. |
| `Esc` | **§6.0**: **도움말 다이얼로그**가 열려 있으면 **닫기만**. 그렇지 않고 최상위가 **필터 입력 포커스**이면 **로그 목록 화면으로 pop**. 그 외는 **§6.5**·**§6.6**. |

### 6.2 공통 이동키

**기본 탐색 모드**·**필터 모드**·**필터 입력 포커스** 모두에서 동일하게 **목록**을 스크롤·페이지한다(필터 입력 중에도 **§6.0**). **`Ctrl+W`**(줄 줄바꿈, §6.1)도 동일하게 **필터 초안에 들어가지 않고** 목록 표시만 바꾼다.

**스크롤·`follow`**: 이 표의 이동키는 입력 시 **`follow`를 끈다**(§5·§8.5). 비확장 **`↑`/`↓`** 로 커서가 **실제로 한 줄 이동**한 뒤, 커서가 **마지막 논리 줄**이고 뷰가 **꼬리 정렬**이면 **`follow`를 다시 켠다**. **마지막 논리 줄**에서 **`↓`** 로 **커서가 움직이지 않을 때**는, **`follow`가 이미 켜져 있고** 꼬리 정렬이면 **`follow`를 끄지 않는다**; **`follow`가 꺼져 있으면** 무동작만으로는 **켜지지 않는다**. **맨 위**에서 **`↑`** 로 커서가 안 움직이면 **`follow`가 꺼져 있을 때** 마찬가지로 **켜지지 않는다**. `PageDown`/`Ctrl+F`는 처리 후 **꼬리 정렬**이면 `follow`를 켠다(이미 마지막 줄에서의 무동작 포함). **§6.3**의 **wrap off** **목록** 가로 스크롤(`←`/`→` 등, **필터 입력 포커스의 `←`/`→` 제외**, §6.4)은 **꼬리 정렬이면 `follow`를 켜고**, 아니면 **끈다**(§8.5).

| 키 | 동작 |
|----|------|
| `↑` / `↓` | 커서 한 줄 위/아래(비확장 이동 시 **연속 구간(앵커)만 해제**, Space pick은 유지). **가시 영역의 맨 위/맨 아래 행**에서 한 줄 더 가면 **뷰만 한 줄** 위/아래로 스크롤한다(§8.5). |

| `Ctrl+F` | `PageDown`과 동일. |

**로그 목록 화면**(`FocusLogList`)에서 **`h`/`j`/`k`/`l`** 은 **`←`/`↓`/`↑`/`→`** 와 각각 동일하고, **`G`** 는 **`End`** 와 동일하다(§6.3 표).

가로 스크롤(**로그 목록 화면**에서의 `←`/`→`, §6.3)·Shift 연속 구간·**`Space`**·**`Ctrl+n`/`Ctrl+p`**·목록에서의 **`c`** 는 **§6.3** 표를 따른다(**필터 입력 포커스**의 `←`/`→`는 §6.4).

### 6.3 탐색·선택·검색 이동(보조·별칭·하이라이트 이동)

**필터·하이라이트 스택**(§8.0): 이동·**`Ctrl+n`/`Ctrl+p` 매칭 탐색**의 **줄 집합**은 항상 **현재 필터 결과**(기본 탐색 모드면 전체)이다. **하이라이트 모드**는 그 위에 겹쳐져 **표시 줄 안에서만** 매칭·강조한다.

**로그 목록 화면**에서 **`h`/`j`/`k`/`l`** 은 **`←`/`↓`/`↑`/`→`** 와 동일하게 **목록**만 스크롤하고, **`G`** 는 **`End`** 와 동일하다(**필터 입력 포커스**·**검색(highlight) 입력**에서는 §6.4·§6.5대로 **초안에 인쇄 문자**로만 반영된다).

**필터 입력 포커스**에서도 아래 키는 **필터 초안에 append 되지 않는다**(단, **`←`/`→`는 §6.4**대로 **초안 캐럿**만 이동). 목록은 **적용 중인 필터(`prog`)** 기준으로 유지되고(초안은 첫 줄 필터 크롬에만 반영), 커서·스크롤·선택만 갱신한다. **`Ctrl+n` / `Ctrl+p`**: **하이라이트 모드**일 때만 다음/이전 매칭 이동이며(**순환 없음**, §8.4), **그 외에는 키만 소비**한다(필터 초안에도 들어가지 않음; **실제로 커서가 바뀐 경우에만** 이동 시 **`follow` 끊김**; 커서가 바뀐 뒤 마지막 줄·꼬리 정렬이면 §8.5대로 **`follow` 재개**).

**로그 목록 화면 — 줄 선택·복사 요약**

- **`Shift+↑` / `Shift+↓`** 등: **연속 구간** 지정·확장(§6.3·§8.6).
- **`Space`**: pick 토글 또는 구간 → pick 일괄 추가(§6.3 표).
- **`c`**: **구간 ∪ pick**(중복 제거, 필터 인덱스 **오름차순**); 둘 다 없으면 **커서 한 줄**; 성공 시 **`N lines copied`** / **`1 line copied`**.
- **`Esc`**: **§6.5**·**§6.6**과 같이 **목록 선택(구간+pick) 해제가 검색(compose) 해제보다 앞선다**.

### 6.4.1 필터·하이라이트 히스토리 오버레이

- **목적**: 이전에 **성공 적용**한 필터 식·**확정**한 하이라이트 검색어를 **목록에서 고르고** 현재 **초안**에 넣는다(확정 적용은 각각 필터 `Enter`·검색 compose `Enter`가 담당).
- **열기**: **필터 입력 모드**에서 **`↓`**(오버레이가 닫힌 상태) → **필터 히스토리** 오버레이. **검색(highlight) 입력 중**에서 **`↓`**(오버레이 닫힘) → **하이라이트 히스토리** 오버레이. **로그 목록 화면**(compose 아님)에서는 **`↓`**는 §6.2·§6.3만 적용한다.
- **표시**: 터미널 중앙 근처 **작은 모달 목록**(항목은 저장된 MRU 순). 항목이 없으면 **빈 목록 안내** 한 줄만 보여도 된다.
- **`↑` / `↓`**: 오버레이 **열림**일 때만 **선택 바(하이라이트)만** 위·아래로 이동; **로그 목록 커서·스크롤은 변하지 않는다**.
- **`Enter`**: 현재 선택 항목 문자열을 **해당 모드의 초안**에 넣고(필터면 **필터 초안**, 검색 compose면 **검색 초안**), **캐럿을 초안 끝**으로 맞춘 뒤 오버레이를 닫는다. **빈 히스토리**에서는 초안을 바꾸지 않고 닫을 수 있다.
- **`Esc`**: 오버레이만 닫는다. **초안·적용 필터·적용 검색어는 변경하지 않는다**.
- **그 외 키**: 오버레이가 열린 동안 **소비만**(초안에 들어가지 않음).

### 6.5 기본 탐색 모드(적용 필터 없음·로그 목록 화면)

**검색(highlight) 입력 아님**: 인쇄 문자·`Backspace`는 **검색 초안에 넣지 않는다**. **`Space`**는 **목록 pick**(§6.3). **`c`**는 목록이 비어 있지 않으면 **항상** 클립보드 복사(**구간 ∪ pick**(오름차순·중복 제거), 둘 다 없으면 커서 한 줄, §6.3·§8.6). 그 밖의 인쇄 문자는 **무시**된다(**단, `m`·`1`–`9`는 §6.7 북마크**). **`/`** 한 번이면 **검색(highlight) 입력 push**: **검색 초안**을 현재 **적용 검색어**로 초기화하고 **캐럿을 초안 끝**에 둔 뒤, 둘째 크롬 줄이 **`HIGHLIGHT  │  > …`** 편집 표시로 바뀐다(§5: 끝 캐럿일 때 `…_`, 중간일 때 `…|…`).

**검색(highlight) 입력 중**:

| 키 | 동작 |
|----|------|
| 인쇄 문자 | **검색 초안**에 **캐럿 위치**에 삽입(하이라이트는 **compose 중 `Enter` 확정 후**만 갱신, §8.1). |
| `Space` | **검색 초안**에 **캐럿 위치**에 공백 삽입 |
| `Backspace` | **캐럿 앞** 한 글자(루 단위) 삭제 |
| `←` / `→` | **검색 초안** 안의 **캐럿**만 좌·우로 이동한다. **로그 목록의 가로 스크롤**(`←`/`→`, §6.3)과는 별도이며, compose 중에는 **초안 편집이 우선**한다. |
| `↓` | **§6.4.1**: 히스토리 오버레이가 **닫혀 있으면** **하이라이트 히스토리**를 연다(로그 목록 `↓`와 구분). **열림**일 때는 §6.4.1 표만 적용. |
| `Enter` | **검색 초안**을 `TrimSpace`한 뒤 **적용 검색어**로 확정하고 **compose pop**. 결과가 빈 문자열이면 하이라이트 모드 해제. 확정 직후 **검색 초안**을 적용 검색어와 동기화한다. |
| `/` | **compose pop**(초안을 **적용 검색어**로 되돌림) 후 **필터 입력 모드**로 전환한다(§8.1; `/` 문자는 검색 초안에 삽입되지 않음). |
| `:` | **`/`와 동일**(compose pop 후 필터 입력 모드; `:` 문자는 검색 초안에 삽입되지 않음). |
| `Esc` | **목록 선택**(연속 구간 또는 Space pick)이 있으면 **그것만 해제**한다(검색(highlight) 입력·적용 검색어·하이라이트는 그대로). **없을 때만** **compose pop**: **검색 초안**을 **적용 검색어**로 되돌리고(편집 취소), **적용 검색어·하이라이트는 변경하지 않는다**. |

**로그 목록 화면·기본 탐색 모드**에서 `Esc`는 **한 번에 한 단계**이며, 다음 **순서**로 처리한다: (1) **목록 선택이 있으면** **연속 구간 + Space pick을 함께 해제**. (2) **그렇지 않고 검색(highlight) 입력 중이면** **검색 입력만 종료**(초안을 적용 검색어로 되돌림, 위 표 `Esc`와 동일). (3) **그렇지 않으면** **적용 검색어·검색 초안 전체 클리어**(하이라이트 모드 해제, `searchCompose`도 해제).

### 6.6 필터 모드(적용 후·로그 목록 화면)

| 키 | 동작 |
|----|------|
| `Esc` | **우선순위**(한 번에 한 단계만): (1) **목록 선택**(연속 구간 + Space pick)이 있으면 **그것만 해제**(적용 필터·적용 검색어·검색 초안·compose 유지). (2) **검색(highlight) 입력 중**이면 **compose pop**만(§6.5 compose 표 `Esc`와 동일: 초안 되돌림, 적용 검색어 유지). (3) 그 외 **적용 검색어·검색 초안 전체 클리어**(하이라이트 모드 해제, `searchCompose` 해제). **`Esc`는 적용 필터·`prog`·필터 모드에 영향을 주지 않는다**; 필터를 바꾸거나 끄려면 **`Enter`** 또는 **`:`**로 필터 입력 모드에 들어가 **§6.4**대로 편집·`Enter` 적용한다(초안을 비우고 적용하면 §6.4와 동일하게 전체 줄 표시). |

**커서 보존**: 필터 입력에서 **빈 식(또는 공백만)** 을 적용해 필터를 끄는 경우, **`Enter` 적용** 시 **§6.4**와 같이 **동일 링 행**을 가리키도록 `cursorIdx`를 맞춘다.

터미널이 보내는 **Shift+화살표/Home/End** 시퀀스에 의존한다. `Shift+PageUp/Down`은 터미널·라이브러리에서 별도 키로 안 올 수 있어 미지원일 수 있다.

### 6.7 북마크(목록 탐색 전용)

**적용 조건**: **필터 입력 포커스가 아니고**, **검색(highlight) 입력이 아닐 때**(`searchCompose == false`)에만 본 절의 북마크 동작이 적용된다. **필터 입력 포커스**에서는 §6.4·§6.3이 우선한다. **검색 compose 중**에는 **`m`·`1`–`9`는 §6.5**의 **검색 초안** 편집에만 쓰인다. **§6.4.1 히스토리 오버레이가 열려 있으면** §6.4.1이 우선한다(북마크 키는 오버레이 규칙에 따름).

**슬롯**: **9개의 고정 슬롯**, 번호 **`1`–`9`**. 각 슬롯은 **비어 있거나** 정확히 하나의 **레코드 시퀀스(`Seq`)** 를 가리킨다(필터 목록 인덱스가 아님). **한 `Seq`는 최대 한 슬롯에만** 존재한다(커서가 이미 북마크된 줄에 있을 때 **`m`** 은 **해제**만 한다). **9칸 가득 시 순환 덮어쓰기**를 위해 구현은 **다음에 덮어쓸 슬롯 번호(1–9)** 를 내부적으로 유지한다(빈 슬롯에 새로 넣을 때는 **가장 작은 빈 슬롯** 규칙만 쓰고 이 포인터는 바꾸지 않는다).

**`m` (토글·할당)**:

- 현재 커서 줄의 **`Seq`가 이미 어떤 슬롯에 북마크되어 있으면**: 그 **슬롯만 비운다**(다른 슬롯의 번호·대상은 **변경하지 않는다**).
- **북마크되어 있지 않으면**: **비어 있는 슬롯**이 하나라도 있으면, 그중 **번호가 가장 작은 빈 슬롯**에 현재 줄의 **`Seq`를 기록**한다(첫 북마크는 `1`, 두 번째는 남은 슬롯 중 최소 번호가 `2`가 되는 식).
- **9개 슬롯이 모두 점유**된 상태에서 **새 줄**에 북마크를 추가하면: **다음에 덮어쓸 슬롯**을 **1→2→…→9→1** 순으로 **순환**한다(첫 번째 추가는 **슬롯 1**, 그다음은 **슬롯 2**, …). **덮어쓰이지 않는 슬롯**의 **`Seq`·표시 번호는 그대로** 둔다. **해제(`m`)만**으로는 이 순환 포인터를 바꾸지 않는다(다음 덮어쓰기 위치는 그대로 진행된다).

**`1`–`9` (점프)**:

- 해당 번호 슬롯에 **`Seq`가 없으면**(빈 슬롯) **무동작**(커서·스크롤 불변, 키는 **소비**).
- 슬롯에 **`Seq`가 있으나** 그 시퀀스가 **현재 필터 목록에 나타나지 않으면**(필터에 걸러짐·`--max-lines`로 링에서 제거됨 등) **무동작**(키 **소비**). 목록에 **표시되지 않는 동안**에는 해당 줄에 **북마크 마커를 그리지 않는다**(슬롯 점유는 유지해도 되고, 구현이 슬롯을 자동으로 비우는 것은 허용되나 **번호 재배열은 하지 않는다**).
- 슬롯의 **`Seq`가 현재 필터 목록에 있으면**: 커서를 **그 줄의 필터 목록 인덱스**로 옮기고, **§8.5**에 따라 스크롤을 맞춘다. **`follow` 끊김·복구**는 **다른 커서 이동**과 동일하게 §8.5·§6.2를 따른다. 이동 시 **연속 구간만 해제**하고 **Space pick은 유지**한다(§6.3·`Ctrl+n`/`Ctrl+p`와 동일 취지).

**표시**: §5 **목록** 항목. 시퀀스·본문 사이 폭은 구현이 **고정 디스플레이 폭**을 확보해(줄번호 모드: 시퀀스 필드 + 공백 + **1칸 배지** + 본문 전 공백; `--no-line-numbers`: **2칸**) 배지·ANSI가 있어도 본문 열이 밀리지 않게 맞춘다.

**영속성**: **`state.json` 등에 저장하지 않는다**. 프로세스 종료 시 소멸한다.

**기타**: 북마크는 **적용 필터·적용 검색어·Space pick·연속 구간**과 독립이다. **필터 `Enter` 적용**·**목록에서 `Enter` 또는 `:`(필터 입력 진입)** 등으로 목록 선택이 해제될 때(§8.6) **북마크는 자동으로 지우지 않는다**.

## 7. 필터 문법

공백으로 구분된 토큰. 큰따옴표로 공백 포함 토큰 허용. **태그 토큰의 `value` 안에 공백이 필요하면** 해당 토큰 전체를 큰따옴표로 감싼다(예: `"over_speed: false"`). **따옴표 없이** `tag:` 로 끝나는 토큰 바로 다음에 오는 토큰이 **`:` 를 포함하지 않으면**, 구현은 둘을 **한 태그 토큰**으로 이어 붙인다(예: `over_speed:` + `false` → `over_speed: false`). 값에 `:` 가 들어가면 한 토큰으로 따옴표 처리한다.

### 7.0 토큰 종류

- **단순 토큰**: 태그 형(아래 정규식)이 **아닌** 토큰.
- **태그 토큰**: `^([+-]?)([A-Za-z0-9_/.\-]+):(.+)$` 에서 **선행 부호가 있거나 없고**, **첫 `:`** 뒤에 **value**가 비어 있지 않은 형태. **선행 `+` / `-` 가 없으면 `+` 로 해석**한다(즉 생략은 포함). (`:` 는 **필터 토큰**에서만 tag·value 구분자이며, 줄 안 구분자는 §7.4.)

### 7.1 로그 포맷(레벨 추출)

- **목적**: 예약 태그 **`level`** 의 value 비교는 **줄에서 추출한 심각도 문자열**을 정규화한 뒤 수행한다.
- **CLI `--log-type`**: **`plain`** 은 구조적 레벨 추출을 하지 않는다(필터 **`level:`** 는 추출 실패로 취급). **`adb`** 는 아래 Android 패턴만 사용한다. **`auto`** 는 **`--log-type-probe-lines`** 개의 **비어 있지 않은** 줄(링에서 **가장 오래된** 순)로 점수를 매겨 **android > bracket** 이면 **Android**, 그 외(**`bracket >= android`** 또는 무득점)는 **plain** 으로 **한 번** 확정한다. 샘플 줄 수가 probe보다 적으면 EOF(입력 종료) 시점에 동일 규칙으로 확정한다.
- **설정 파일(`config.json`)**: `log_type.default`, `log_type.probe_lines`, `log_type.patterns`(regex)를 지원한다. 우선순위는 **CLI > config > 기본값**이다. `state.json`은 히스토리 전용으로 유지한다.
- **포맷 판별(구현 참고)**: 샘플 줄마다 **브라켓 패턴**이면 **bracket 점수 +1**, **Android 패턴**이면 **android 점수 +1**.
- **브라켓 줄**: `^\[[^\]]+\]\s+([A-Za-z][A-Za-z0-9_]*)\s*:` 로 앵커할 때, 첫 캡처 그룹이 **레벨 토큰**(예: `INFO`, `error`). 뒤이어 오는 `:` 까지가 한 덩어리.
- **Android 줄**(logcat `-v time` 또는 `-v threadtime`): `-v time`은 `^\d{4}-\d{2}-\d{2}\s+\d{1,2}:\d{2}:\d{2}\.\d{3}\s+\d+-\d+\s+\S+\s+\S+\s+([VDIWEF])\s`, `-v threadtime`은 `^\d{2}-\d{2}\s+\d{1,2}:\d{2}:\d{2}\.\d{3}\s+\d+\s+\d+\s+([VDIWEF])\s+\S+:` 형태를 지원하며, 한 글자 캡처가 레벨 문자다. **`level:DEBUG`** 같은 필터 value는 **`D`** 와 매칭되도록 **V/D/I/W/E/F → 풀네임**(VERBOSE, DEBUG, INFO, WARN, ERROR, FATAL)으로 정규화해 비교한다.
- **`FormatUnknown`(내부) / 레거시 한 줄 평가**: **`FormatUnknown`** 은 한 줄에서 **브라켓 먼저**, 실패 시 **Android**를 시도한다. 둘 다 실패하면 **추출 레벨 없음**. (**`--log-type=auto`** 확정 결과는 **`plain`** 이면 구조적 추출을 하지 않는다.)

`--ignore-case`가 켜져 있으면 **필터·태그** 일치에 쓰이며, **level value 비교**에도 **대소문자 무시**로 적용한다.

### 7.2 결합 규칙(AND / OR / 나열 순서)

1. **단순 토큰**(`+term`, `-term`, `term`): 기존과 같이 줄 **전체**에 대한 **부분 문자열 포함/제외**이며, 토큰끼리는 **AND**.
2. **태그 토큰** `+tag:value` / `-tag:value`(또는 부호 생략으로 `+` 처리): **서로 다른 `tag` 키**(대소문자 무시하여 동일 키로 묶음)에 대한 조건은 **AND**.
3. **동일 `tag` 키**에 여러 토큰이 있을 때: 필터 문자열 안에서 **토큰을 왼쪽부터 순서대로** 읽어 적용한다. **동일 `(tag, value)` 쌍**에 대해 `+`와 `-`가 **둘 다** 나오면 **먼저 나온 토큰만 유효**하고, **같은 쌍을 다시 지정한 뒤쪽 토큰은 무시**한다. 그렇게 확정된 각 `value`마다 **`+` 이면 포함 후보**, **`-` 이면 제외 후보**이다. **포함 후보 value 들**은 그 태그 축에서 **OR**(하나라도 맞으면 포함 조건 충족). **제외 후보 value 들**은 **OR로 제외**(어느 하나와 맞으면 줄 제외). **포함 후보가 비어 있지 않으면** 줄은 **그 OR 중 하나**를 만족해야 하고, **제외 OR**에 걸리면 **항상 제외**.
4. **예약 `level` 태그**: 줄에서 **§7.1**에 따라 추출·정규화한 심각도와 필터 **value**만 비교한다(줄에 `level:` 문자열이 없어도 됨). **추출 실패**인 줄은, 해당 축에 **포함(`+`) value가 하나라도 있으면 통과하지 않는다**. 제외(`-`)만 있는 축에서는 **추출 실패 줄은 제외 value와 일치하지 않은 것으로 본다**(통상 통과).
5. **`level` 이 아닌 태그**: **§7.4** KV 동치 규칙으로 판단한다.
6. **최상위 OR (`|`)**: 토큰화·`tag:`+값 병합(§7 도입) **이후**, **토큰이 정확히 `|` 인 위치**에서 식을 **왼쪽부터** **가지(branch)** 로 나눈다. **빈 가지**(선행·후행·연속 `|`)는 파싱 오류다. **각 가지**는 **독립된 부분 식**으로 §7.0–§7.2를 적용한다(가지 안: plain·서로 다른 태그 키는 **AND**; 동일 태그 키의 다중 `+` 값은 **OR**). **줄은 한 가지라도 만족하면** 전체 필터를 통과한다. **`||` 를 공백 없이 붙이면** 한 토큰이 되어 **오류**로 처리한다(공백으로 `|` 를 분리할 것).
7. **`tag:` 값 병합과 `|`**: §7 도입의 `tag:` + 다음 토큰 병합은, 다음 토큰이 **OR 구분자 `|`** 이면 **수행하지 않는다**(그렇지 않으면 `|` 가 값에 흡수되어 OR가 사라짐).

### 7.3 태그 형 문법(리마인더)

- `tag`는 `[A-Za-z0-9_/.\-]+`(필터 입력에서의 키 이름). `value`는 **필터 토큰**에서 **첫 `:`** 이후 전부(값 안에 `:` 허용).
- **필터에서 `tag` 와 첫 `:` 사이에는 공백·탭을 두지 않는다**(지원하지 않음). `over_speed : false` 처럼 쓰면 공백 토큰 분리로 태그 토큰이 되지 않는다. 값 앞 공백만 필요하면 `over_speed: false` 또는 `over_speed:` + `false` 병합(§7 도입) 또는 `"over_speed: false"` 를 쓴다.
- 태그 토큰 판별은 **7.0** 정규식을 따른다. `+` 단독 등 불완전 토큰은 파싱 오류.

### 7.4 일반 태그 KV 동치(`level` 제외)

- **목적**: 로그에 `key=value`, `key: value`, `key:value` 형태가 섞여 있어도, 필터는 **`tag:value`** 토큰만으로 동일 키·값을 찾는다.
- **키 경계**: 줄에서 `tag` 문자열을 찾을 때, 바로 **앞** 문자가 **§7.3** `tag` 문자 집합에 속하면 **그 위치는 매칭하지 않는다**(더 긴 식별자의 접미로 오인 방지). 줄 시작이면 허용.
- **구분자(로그 줄)**: 키 **끝부터** 구분자까지 **앞쪽 공백·탭은 무시**하고, **`=` 또는 `:`** 하나를 구분자로 인정한다. 구분자 **뒤**도 공백·탭을 무시한 뒤 값을 읽는다. 즉 `key = value`, `key : value` 도 **§7.3** 필터 규칙과 동치 비교한다.
- **값 추출**: 구분자 뒤(앞뒤 공백·탭 trim)부터 **첫 콤마(`,`)**, **첫 유니코드 공백**, 또는 **줄 끝** 전까지를 값으로 한다. 값·필터 `value`는 비교 전 **`TrimSpace`**. `--ignore-case`면 키 위치 탐색·값 비교에 **대소문자 무시**를 적용한다.
- **한계**: 값 안에 콤마·공백을 넣어야 하면 본 PRD 범위 밖(따옴표 규칙 등은 추후).

파싱 실패 시: 상태바 오류, **이전 유효 필터 유지**.

### 7.5 태그·plain 조합 예시(요약)

- **`timeout retry`**: 줄에 `timeout` **과** `retry` 부분 문자열(plain AND).
- **`level:ERROR level:WARN`**: 추출 레벨이 ERROR **또는** WARN(동일 키 `level` 내 OR).
- **`level:ERROR timeout`**: 레벨 ERROR **이고** 줄에 `timeout` 포함(태그·plain AND).
- **`level:WARN` + 공백 + 파이프(`|`) 토큰 + 공백 + `timeout`**: 레벨 WARN **이거나** 줄에 `timeout` 포함(최상위 가지 OR). `|` 는 **공백으로 분리된 단독 토큰**이어야 한다.
- **공통 태그를 OR 가지마다 걸기**: `service:prod err` + 파이프 + `service:prod warn` 처럼 **태그를 각 가지에 반복**한다. `service:prod err` + 파이프 + `warn` 만 쓰면 둘째 가지에는 `service:prod` 조건이 없다.

## 8. 검색·하이라이트

### 8.0 필터와 하이라이트의 스택(stacking)

- **하이라이트 모드**는 로그를 보는 **기본 탐색 모드**(적용 필터 없음)와 **필터 모드**(적용 필터 있음) **모두**에서 동작한다. 두 모드는 하이라이트와 **배타적이 아니라** 아래 순서로 **쌓여** 적용된다.
- **레이어 1 — 목록(필터)**: 적용 필터가 있으면 `prog` 조건을 통과한 줄만 목록에 남긴다. **기본 탐색 모드**에서는 필터가 없으므로 **링의 전체 줄**이 목록이다.
- **레이어 2 — 하이라이트·매칭 이동(`n`/`p`·`Ctrl+n`/`Ctrl+p`)**: **적용 검색어**가 비어 있지 않으면(**하이라이트 모드**), **레이어 1에 남은 줄들의 가시 텍스트**에만 하이라이트를 그린다. **`n`/`p`(로그 목록)·`Ctrl+n`/`Ctrl+p`의 매칭 이동**도 **그 목록의 인덱스** 범위 안에서만 이루어지며 **한 방향으로만 스캔**(목록 끝·처음에서 순환하지 않음; 필터 밖에 있지만 적용 검색어에만 맞는 줄은 목록에 없으면 스킵).
- **축의 독립성**: 필터 식은 **적용 검색어·검색 초안**을 바꾸지 않고, 검색 입력은 필터 `prog`를 바꾸지 않는다. 다만 **보이는 결과**는 위 레이어 순서의 합성이다.

### 8.1 검색 초안과 적용 검색어

- **검색(highlight) 입력**: **로그 목록 화면**에서 **첫 `/`**로 켜지고, **`Enter` 또는 `:`로 필터 입력 모드에 들어가면** compose는 끄고 **검색 초안**을 **적용 검색어**로 되돌린다(필터 스택이 우선). **필터 `Enter`로 적용**해 목록으로 돌아올 때도 compose는 꺼진 상태다.
- **검색 초안**: **검색(highlight) 입력 중·목록**에서만 편집한다. **Unicode `IsPrint`**인 문자만 **캐럿 위치**에 삽입한다(제어 문자 제외). `Space`·`Backspace`는 **캐럿 기준**으로 동작하며(§6.5), **`←`/`→`는 캐럿만** 움직인다. **§6.2·§6.3**의 **세로·페이지 이동** 등은 compose 중에도 **목록**에 적용되고, **`←`/`→`만** compose가 가로챈다(목록 가로 스크롤과 배타). 매칭 이동은 **`Ctrl+n`/`Ctrl+p`**(compose 중 **`n`/`p`는 초안 문자**이므로 §6.3·§8.4). **compose 중 `c`**는 검색 초안에 삽입되며, **목록·compose 아님**일 때만 **`c`가 복사**(§8.6)다.
- **필터 초안**: **필터 입력 포커스**에서만 편집한다(§6.4). **`←`/`→`는 캐럿**(UTF-8 루 단위)만 이동하며 **목록 가로 오프셋(`colRuneOff`)과 배타**다. **§6.2·§6.3**의 **세로·페이지 이동** 등은 필터 입력 중에도 **목록**에 적용되고, **`←`/`→`만** 필터 초안이 가로챈다(§6.3의 목록 가로 스크롤과 배타). 매칭 이동은 여전히 **`Ctrl+n`/`Ctrl+p`**(하이라이트 모드일 때; **`n`/`p`는 필터 초안 문자**).
- **compose 중 `Enter`**: **검색 초안**을 `strings.TrimSpace`한 값으로 **적용 검색어**에 반영하고 compose를 끈다. **그 전까지** 목록 **하이라이트는 적용 검색어만** 따르며, compose 중 초안만 바뀐 상태에서는 하이라이트가 바뀌지 않는다.
- **compose 중 `Esc`**: **목록 선택**(연속 구간 + Space pick)이 있으면 **그것만 해제**한다. **없으면** **검색 초안**을 **적용 검색어**로 되돌리고 compose만 끈다(하이라이트 불변).
- **하이라이트 모드**: **적용 검색어**가 비어 있지 않으면 **§2**와 같다. **`n`/`p`(로그 목록)·`Ctrl+n`/`Ctrl+p`**만 매칭 줄 이동(**필터 입력 포커스**에서는 **`Ctrl`만**; 적용 검색어가 있으면 동일).
- **필터와의 관계**: **§8.0 스택**과 동일. **적용 검색어**는 **필터를 통과한 뒤 남은 줄**(또는 필터 없음 시 전체)의 가시 텍스트에만 하이라이트를 적용한다.

### 8.2 매칭 규칙(하이라이트 = 하이라이트 모드에서 `n`/`p`·`Ctrl+n`/`Ctrl+p` 이동 기준)

- **적용 검색어**가 **빈 문자열**이면 **하이라이트 모드 아님**·하이라이트·**`n`/`p`·`Ctrl+n`/`Ctrl+p` 매칭 이동 없음**(해당 키는 소비만).
- **다중 토큰**: 확정된 적용 검색어는 **필터와 동일한 토큰 분리 규칙**으로 **검색 토큰** 목록을 만든다: **ASCII 공백·탭**으로 구분; **큰따옴표 `"…"`** 로 감싼 구간은 **내부 공백을 유지한 하나의 토큰**이다(§7 `Tokenize`와 동일). **따옴표 짝이 맞지 않으면**(미닫힘 등) 토큰화 오류로 보고, **토큰 하나**로서 **적용 검색어 전체 문자열**을 그대로 바늘로 쓴다.
- **줄 매칭(`n`/`p`·`Ctrl+n`/`Ctrl+p`)**: 한 줄이 **토큰 중 하나라도** 위 **부분 일치**를 만족하면 그 줄은 **매칭 줄**이다(**OR**).
- **화면 강조**: 한 줄 안에서 **각 토큰**에 대해 위와 같은 **모든** 부분 일치 구간을 강조한다. 구간이 **겹치면** 하나의 연속 구간처럼 합쳐 그린다.
- **대소문자(하이라이트 전용)**: 하이라이트·**하이라이트 모드**의 **`n`/`p`·`Ctrl+n`/`Ctrl+p` 매칭**은 **항상 대소문자를 구분**한다. 줄 안에서 **각 토큰 문자열**과 **바이트 단위** `strings.Index` 동등의 **부분 문자열** 일치(UTF-8 바이열; 토큰·가시 슬라이스가 모두 유효 UTF-8일 때 기대대로 동작). **`--ignore-case`는 하이라이트에 적용하지 않는다**(필터·태그만 제어).
- **부분 일치**: 위 **대소문자 구분** 기준으로 줄의 **부분 문자열** 일치.
- **가로 스크롤(wrap off)**: 화면에 그리는 것은 가로 오프셋 이후의 **가시 슬라이스**뿐이며, 하이라이트는 그 **가시 슬라이스 문자열**에 대해 적용된다(스크롤로 잘린 앞부분은 해당 뷰에서 강조하지 않음).
- **줄 줄바꿈(wrap on, §6.1)**: 각 **시각 행**에 그리는 텍스트는 해당 **랩 구간**뿐이며, 하이라이트·가시 텍스트 판단도 **그 구간 문자열**에 적용된다.

### 8.3 시각적 표현

- 일치 구간: 터미널에서 구분되도록 **배경색·전경색**(lipgloss 등)으로 강조.
- **커서 줄**: 목록에서 포커스인 **논리 줄**은 **Reverse** 등으로 추가 강조한다(**`follow` on/off**와 무관). 검색 하이라이트·연속 구간·Space pick 표시는 각각 §8.1·§8.6 규칙을 유지한다.
- **연속 구간·pick 줄**: 연속 구간과 Space pick만 해당하는 줄은 배경 톤으로 구분(겹치면 연속 구간 스타일 우선); 커서와 겹치면 스타일이 겹칠 수 있다.

### 8.4 하이라이트 모드: 매칭 줄 이동 (`n` / `p` · `Ctrl+n` / `Ctrl+p`)

- **하이라이트 모드**(§2·§8.1)에서만 **이동**이 일어난다. 즉 **적용 검색어**가 **비어 있지 않을 때** **`n`/`p`**(**로그 목록 화면** `FocusLogList` 전용, `h`/`j`/`k`/`l`과 같은 remap 방식) 또는 **`Ctrl+n`/`Ctrl+p`**(**`tryBrowseKey`**, 필터 입력 포커스 포함)가 **이동 키**다. **필터 입력 포커스**에서는 **`n`/`p`는 필터 초안에 인쇄 문자**로 들어가므로, 매칭 이동은 **`Ctrl+n`/`Ctrl+p`**만 사용한다. **적용 검색어가 비어 있으면** 해당 키는 **소비만**한다. 스캔 범위는 **§8.0 레이어 1**에 올라온 줄들(기본 탐색 모드 = 전체, 필터 모드 = 통과 줄만)의 **필터 목록 인덱스**로 한정되며, **목록 끝에서 맨 반대 끝으로 이어 붙이는 순환(wrap)은 하지 않는다**.
- **다음(`n` 또는 `Ctrl+n`)**: 현재 커서의 **필터 목록 인덱스보다 큰 인덱스**만 위에서 아래로 스캔하여, 하이라이트와 **동일 조건으로 매칭하는 첫 줄**이 있으면 그 줄로 커서를 옮긴다. **그보다 아래에 매칭이 없으면** 커서는 **그대로**다(키는 소비됨, `tryBrowseKey` 규칙과 동일).
- **이전(`p` 또는 `Ctrl+p`)**: 현재 인덱스보다 **작은 인덱스**만 아래에서 위로 스캔하여, 동일 조건으로 매칭하는 **가장 가까운 이전 줄**로 옮긴다. **위쪽에 매칭이 없으면** 커서 **유지**.
- “매칭”의 정의는 **8.2**와 동일(화면 하이라이트와 같은 조건).
- 이동 시 **연속 구간(앵커)만 해제**하고 **Space pick은 유지**한다(§6.3).

### 8.5 커서·뷰 정렬

- 커서는 항상 뷰포트 안에 들어오도록 스크롤을 조정한다: **wrap off**일 때는 논리 줄 인덱스 기준 `scrollTop`, **wrap on**일 때는 **시각 행(랩 세그먼트)** 인덱스 기준 `scrollSegTop`이다. 커서 인덱스(`cursorIdx`)는 **항상 논리 줄(필터 목록 인덱스)** 이다.
- **최소 스크롤(논리 줄, wrap off)**: `scrollTop`은 커서가 `[scrollTop, scrollTop+vh)` 안에 들어오는 한 **유지**한다. 커서가 그 **위로** 벗어나면 `scrollTop = cursorIdx`(한 줄씩 위로 밀림). **아래로** 벗어나면 `scrollTop = cursorIdx - vh + 1`. 이후 `scrollTop`을 `[0, len(fidx)-vh]`로 클램프하고, 클램프 뒤에도 커서가 가시 영역 밖이면 한 번 더 보정한다.
- **최소 스크롤(wrap on)**: 현재 커서 논리 줄이 차지하는 랩 세그먼트 구간 `[firstSeg, lastSeg]`가 `[scrollSegTop, scrollSegTop+vh)`와 **겹치지 않을 때만** `scrollSegTop`을 바꾼다(위로 벗어남·아래로 벗어남 각각 한 칸씩 맞추는 동등 규칙으로 보정 후 `scrollSegTop`을 `[0, S-vh]`에 클램프). **예외**: **`follow`가 on**이고 커서가 **마지막 논리 줄**이면(시각 행 수 `S`>뷰 높이 `vh`일 때) **꼬리 정렬**을 위해 **`scrollSegTop = S−vh`** 로 두어, 최소 스크롤만으로는 커서가 보이나 시각 꼬리가 아닌 경우를 막는다. **`follow`가 off**인 채 **마지막 논리 줄로 새로 들어온 직후**(비확장 ↑/↓·`PageDown`/`Ctrl+F`·다음 매칭 등)에도 **`follow` 재개·꼬리 정렬** 판정 전에 동일하게 시각 꼬리를 맞춘다.
- **논리 줄** 수 ≤ 뷰 높이일 때(wrap off): 위쪽을 빈 행으로 채워 **전체 목록을 아래에 붙여** 그리며, **마지막 화면 행 = 마지막 로그 줄**이 되게 한다.
- **시각 행** 수 ≤ 뷰 높이일 때(wrap on): 위쪽을 빈 행으로 채워 동일하게 **마지막 화면 행 = 마지막 시각 행**(즉 목록 꼬리)이 되게 한다.
- **`follow` 끊김·복구**: **커서 인덱스가 바뀌는** 목록 조작(§6.2·§6.3·하이라이트 **`n`/`p`·`Ctrl+n`/`Ctrl+p`로 실제 이동이 있을 때** 등)마다 먼저 **`follow`는 off**다(§6.3: 매칭 이동이 **무동작**이면 `follow`는 건드리지 않음). 비확장 **`↑`/`↓`** 는 커서가 **실제로 이동한 경우**, 이동 후 **마지막 논리 줄**이고 **꼬리 정렬**이면 **`follow` on**. **마지막 논리 줄**에서 **`↓`** 로 **이동이 없을 때**는 **`follow`가 이미 on**이고 **꼬리 정렬**이면 **`follow`를 끄지 않는다**. **`PageDown`/`Ctrl+F`**: 처리 후 **꼬리 정렬**이면 **`follow` on**(마지막 줄에서 무동작 포함). **`wrap off`**에서 **로그 목록 화면**의 §6.3 **`←`/`→`·`Shift+←`/`Shift+→`**로 **가로 스크롤**한 뒤 **`follow`를 꼬리 정렬 여부에 맞춘다**(꼬리면 on). **필터 입력 포커스**의 `←`/`→`(§6.4, **`colRuneOff` 불변**)는 이 “가로 스크롤”에 **포함하지 않는다**. **`wrap on`**일 때 §6.3의 이 키들은 오프셋을 바꾸지 않으므로 **`follow`는 건드리지 않는다**. **`End`/`Shift+End`**: 항상 **`follow` on**.
- **마지막 줄 + 신규 입력**: `follow`가 on이고 꼬리 정렬일 때, 버퍼에 줄이 추가되면 **즉시** 커서를 **새 마지막 논리 줄**로 옮기고 `scrollTop` 또는 `scrollSegTop`을 맞춰 **자동 스크롤**한다.

### 8.6 줄 선택·클립보드(하이라이트와 연동되는 탐색 UX)

- **Shift+↑ / Shift+↓**, **Shift+Home / Shift+End**: 앵커(첫 Shift 이동 시점의 커서)와 현재 커서 사이의 **연속 줄 구간**을 표시한다.
- **`Space`**: **로그 목록 화면**·**검색(highlight) 입력 아님**일 때, **연속 구간이 있으면** 구간 내 **모든 줄**을 **Space pick**에 넣고 **연속 구간만 해제**; **없으면** 현재 커서 줄을 pick에서 **토글**한다.
- **`c`**: **로그 목록 화면**·**검색(highlight) 입력 아님**·목록이 비어 있지 않을 때: **연속 구간**에 속한 줄과 **Space pick**된 줄의 **합집합**(동일 필터 목록 인덱스는 **한 번만**)을 필터 목록 인덱스 **오름차순**으로 개행 연결해 **시스템 클립보드**에 복사한다. **구간도 없고 pick도 없으면** **커서 한 줄**만 복사한다. 대상은 **현재 필터 목록**·**링 버퍼** 텍스트(macOS: `pbcopy`, Linux: `wl-copy` 또는 `xclip`). 성공 시 **`N lines copied`** / **`1 line copied`**, 빈 목록이면 **`no lines`**, 그 외 실패 시 **오류 한 줄**(토스트 정책 §6와 동일).
- **`Esc`**: **§6.5**·**§6.6**과 동일. 요약: **목록 선택(구간+pick) 해제 →(없으면) compose pop → 적용 검색어·초안 클리어**; **필터 모드에서도 `Esc`는 적용 필터를 건드리지 않는다**(§6.6). **필터 입력 포커스**의 `Esc`는 **§6.0·§6.4**.
- **비확장 세로·페이지·Home/End·가로 스크롤·`n`/`p`·`Ctrl+n`/`Ctrl+p`** 등은 **연속 구간만 해제**하고 **pick은 유지**한다. **로그 목록에서 `Enter` 또는 `:`(필터 입력 진입)**·**필터 `Enter` 적용 성공** 등은 **연속 구간 + pick 모두 해제**(적용 검색어·검색 초안은 별도 규칙).

### 8.7 성능

- 매우 긴 줄에서는 렌더 비용이 커질 수 있음. **버퍼 상한**은 `--max-lines`로 제한한다.
- **고속 stdin**: 기본 `--stdin-batch-ms`(예: 40ms) 동안 들어온 줄을 한 번에 **UI에만** 반영해 **프레임당 Update 횟수**를 줄인다. `0`이면 UI도 줄 단위 즉시 반영. **`--out`에는 배치하지 않는다**(stdin일 때 줄마다 즉시 기록·Sync).

- stdin 고속 시 **`--stdin-batch-ms`** 로 UI 배치 갱신(기본값 사용, `0`으로 비활성화 가능).
- 파서·매칭·링 버퍼: 유닛 테스트.
