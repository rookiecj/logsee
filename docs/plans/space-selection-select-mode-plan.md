# Space 기반 줄 선택·select mode 계획

PRD 기준 문서: [stdio-log-viewer-prd.md](stdio-log-viewer-prd.md). 본 문서는 구현 전 **요구사항·모델 합의**를 고정한다.

## 목표

- **로그 목록 화면**에서 `Space`로 **현재 커서 줄**의 선택을 토글한다.
- **선택 모드(select mode)** 와 **선택영역 설정 모드(range setup mode)** 를 **구분**한다.
- `Esc`는 기존 PRD와 동일하게 **한 단계씩 pop**(도움말 → 필터 포커스 → 목록에서 선택 해제 → compose → 검색 클리어 등, [stdio-log-viewer-prd.md](stdio-log-viewer-prd.md) §6.0·§6.5·§6.6).
- 상태바에 **선택 관련 정보** 표시(개수 등, 아래 표기 규칙).
- **Shift+↑/↓** 등으로 잡은 **연속 구간**과 `Space`로 고른 **줄 집합**을 함께 지원하되, 키·포커스 규칙을 명확히 한다.

---

## 1. 선택 모델 (합의: OK)

- **연속 구간(선택영역)**: 기존과 같이 `Shift+↑/↓`, `Shift+Home/End`로 **앵커~커서** 구간을 표현한다. 구현은 기존 `selAnchor` + `cursorIdx` 패턴을 유지하거나 동등하게 유지한다.
- **줄 집합(Space pick)**: 필터 목록 인덱스 기준으로 **비연속** 줄을 `Space`로 토글한다. 별도 자료구조(예: `map[int]struct{}` 또는 정렬된 슬라이스)로 보관한다.
- 두 표현은 **동시에 존재할 수 있다**. **복사(`c`)** 는 **연속 구간 ∪ Space pick** 의 **합집합**(필터 목록 인덱스 기준 **오름차순**, 동일 줄 **한 번**)으로 고정한다. **렌더링**은 기존 PRD(§5·§8.6): 겹치면 **연속 구간 스타일 우선** 등 시각 규칙을 따른다.

---

## 2. 선택 모드 vs 선택영역 설정 모드 + 일반 ↑/↓

| 개념 | 의미 |
|------|------|
| **선택영역 설정 모드** | `Shift+↑/↓` 등으로 **앵커가 잡힌 뒤** 연속 구간을 조정 중인 상태(기존 범위 선택 UX). |
| **선택 모드(select mode)** | “선택된 로그”가 **하나 이상** 있는 UI 상태. 여기서 **선택된 로그** = (연속 구간이 비어 있지 않음) **또는** (Space로 고른 줄 집합이 비어 있지 않음). 구현은 파생 상태로 둘 수 있다. |

**일반 `↑`/`↓`(비활장 이동)**:

- **선택영역 설정 모드**에서(즉 Shift로 앵커가 설정된 상태에서) 일반 `↑`/`↓`로 커서를 옮기면 **선택영역(앵커 기반 연속 구간)을 clear**한다.
- **Space로 고른 줄 집합**은 이 동작으로 **지우지 않는다**(일반 이동은 “구간 잡기”만 취소).

이렇게 하면 Shift로 잡은 **임시 구간**과 `Space`로 쌓은 **고정 픽**이 분리된다.

## 3. `Space` 키 — 포커스별 동작 (global 금지)

| 포커스 | `Space` 동작 |
|--------|----------------|
| **로그 목록 화면** (`filterEdit == false` 이고 **검색(highlight) 입력 중 아님**) | 현재 커서 줄에 대해 **Space pick 토글**(선택 모드 진입/유지/해제는 집합·구간 유무로 파생). |
| **검색(highlight) 입력 중** | **검색 초안에만** 입력. 로그 목록·줄 선택 상태에는 **영향 없음**. |
| **필터 입력 모드** | 기존과 같이 **필터 초안**에만 입력. |

정리: `Space`는 **전역 단축키가 아니라** 각 입력 포커스의 텍스트/목록 규칙을 따른다.

---

## 4. `Esc` (합의: PRD 스택 유지)

- 목록 포커스에서의 `Esc` **다단계 순서**는 [stdio-log-viewer-prd.md](stdio-log-viewer-prd.md) §6.5·§6.6과 동일하게 유지한다.
- **첫 단계(선택만 해제)** 에서는 **연속 구간(앵커)과 Space 집합을 한 번에 모두 비운다**. 그다음 `Esc`부터는 기존과 같이 compose pop → 적용 검색어·초안 클리어 등으로 진행한다.

---

## 5. `buildCopyText` / 목록 포커스 `c` (갱신)

| 조건 | 복사 내용 |
|------|-----------|
| **연속 구간** 활성 **또는** **Space pick** 1줄 이상(또는 **둘 다**) | 구간에 속한 인덱스와 pick 인덱스의 **합집합**; 각 논리 줄 **한 번만**, 필터 목록 인덱스 **오름차순**으로 개행 연결. |
| 둘 다 없음 (`Shift` 앵커 없음 · pick 비어 있음) | **커서 한 줄**만 복사. |

**검색 입력 중** `c`는 기존대로 초안 편집. **필터 입력 중** `c`는 필터 초안 편집.

---

## 6. 렌더링 (합의: OK)

- 연속 구간 + Space 집합 각각에 대해 **시각적 구분**이 가능하도록 한다(색/패턴은 기존 목록 스타일에 맞춤).
- `lineSelected` 등 단일 구간 API는 **집합·구간 병행**을 반영하도록 확장한다.

---

## 7. 상태바

- **선택된 로그 개수**를 표시한다.
  - 권장: `picked:N`(Space 집합 크기)와 기존 `sel:lo-hi`(연속 구간)를 **동시에 쓸 수 있으면** 둘 다 짧게 표기, 아니면 합산 `sel:N` 한 가지만 쓰고 PRD §5 예시는 갱신한다.
- 구현 후 [stdio-log-viewer-prd.md](stdio-log-viewer-prd.md) §5 상태바 bullet과 맞춘다.
