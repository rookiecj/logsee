# 화면 레이아웃: 필터 상단 · 상태 하단

## 목표

- **상단**: 필터 입력/표시 영역을 고정하고, 터미널에서 한눈에 들어오도록 **시인성**을 높인다.
- **하단**: 버퍼/입출력/follow 등 **상태 정보**만 모은 **상태 바**를 둔다.
- **중앙**: 나머지 높이는 전부 **로그 뷰포트**(기존 스크롤·랩·커서 규칙 유지).

## 구역 정의 (위 → 아래)

| 구역 | 줄 수 | 내용 |
|------|-------|------|
| 필터 크롬 | 1 | 적용 필터 문자열, **편집 모드**(`FILTER  │  > …`) 시 드래프트·캐럿, 파싱 `err:` (필터 전용). 배경·대비·굵기로 리스트 본문과 구분. |
| 하이라이트·힌트 크롬 | 1 | `HIGHLIGHT  │  …` 패턴(필터 줄과 동형), `> …_` compose, `∅` 미적용. 필터 편집 중에는 짧은 키 안내만 표시. |
| 로그 뷰포트 | `height - 3` | `viewportH()` — 필터·검색 2줄 + 상태 1줄을 제외한 높이. |
| 상태 바 | 1 | `lines`, `in`, `follow`, `wrap`, `out`, 클립보드 플래시, 선택 범위 등. `follow:` / `wrap:` 값은 `on`은 강조색, `off`는 회색 톤. 필터 파싱 오류는 상단 필터 줄로만 노출(중복 방지). **`F1`** 도움말은 전면 중앙 모달(버전·키맵); 본 레이아웃 위에 겹친다. |

## 시인성 (필터 줄)

- **비편집**: 어두운 배경 + 밝은 전경, `FILTER` 레이블로 역할 고정.
- **편집(필터 입력 모드)**: 더 높은 대비(예: 다른 배경색)로 “입력 모드”임을 즉시 구분.
- 전체 폭 `Width`에 맞춰 한 줄로 렌더(과도한 길이는 셀 단위 절단).

## 구현 메모

- `viewportH() = height - topChromeLines - bottomChromeLines` (`topChromeLines=2`, `bottomChromeLines=1`).
- `View()` 출력 순서: `필터` → `검색` → `본문` → `상태`.

## 키맵·동작

- 키맵은 [stdio-log-viewer-prd.md](stdio-log-viewer-prd.md) §6·§8을 따른다(로그 목록: **`Enter`** 필터, **`/`** 하이라이트 입력 등). 레이아웃만 본 문서 범위.
